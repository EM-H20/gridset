# Gridset

## 0. 변경 요약 (v1 → v2)

v1은 "선을 긋는 자유 그리드"에 집중했으나, 시장 조사 결과 **setLog의 진짜 DNA는 "편집 단계의 제거"** 임이 드러남. 경쟁 앱(PhotoGrid, Pic Stitch, Fotor, GridArt) 대부분은 "템플릿 → 조정"이거나 "자동 완성"에 머물러 있어, **두 축을 결합한 "자동 제안 + 자유 조정" 포지션이 비어있음**. v2는 이 틈새를 노린다.

**핵심 변화 3가지**

1. 진입 플로우 역전: "빈 캔버스 → 선 긋기"가 아니라 "사진 고르기 → 자동 레이아웃 제안" 이 첫 경험
2. "선 긋기"는 필수가 아닌 **선택적 고급 기능**으로 재배치 (고급 사용자용 자유도)
3. setLog식 "편집 0" 철학을 전체 UX에 흡수 (자동 영상 싱크, 자동 크롭, 자동 밸런싱)

## 1. 프로젝트 개요

### 프로젝트명

Gridset

### 한 줄 정의

사진·영상을 고르면 그리드가 자동으로 제안되고, 원하면 선을 끌어 자유롭게 조정해 한 장의 이미지 또는 영상으로 내보내는 "편집 없는" 콜라주 앱.

### 태그라인 후보

- "고르기만 해. 나머진 자동이야."
- "자동인데 자유로운 그리드"
- "편집은 생략, 감각은 유지"

### 문제 정의 (Problem Statement)

- **setLog**는 N분할 브이로그를 자동 완성해 큰 인기를 얻었지만, **정각마다 강제 촬영**이라는 제약과 **레이아웃 커스터마이징 불가**라는 한계가 있다.
- 반면 **기존 콜라주 앱(PhotoGrid, Pic Stitch, Canva)**은 수백 개 템플릿을 제공하지만, 본질적으로 **"프리셋 고르기 → 수동 배치"** 흐름이라 setLog의 "한큐" 경험을 주지 못한다.
- 사용자는 **"사진만 주면 알아서 조립되는 자동화"** 와 **"마음에 안 들면 손볼 수 있는 자유도"** 를 동시에 원한다 — 지금은 둘 중 하나를 포기해야 한다.
- 특히 사진과 영상을 한 화면에 섞으려면 대부분의 앱에서 프리셋이 제한되거나, 영상 길이 동기화를 사용자가 직접 맞춰야 한다.

### 제품 목표 (Goals)

1. 사진·영상을 고르기만 하면 **3초 안에** 완성된 레이아웃 3가지를 자동 제안
2. 마음에 안 들면 **선을 끌어** 1초 만에 셀 크기 조정 가능 (프리셋 앱이 못 주는 자유도)
3. 영상이 섞여도 **자동 싱크·자동 크롭**으로 사용자가 편집 단계를 거치지 않음
4. 9:16, 1:1, 4:5, 16:9 등 원하는 비율로 **이미지/영상 포맷 한 번에 저장**
5. MVP는 로그인·클라우드 없이 로컬에서 완결

### 비목표 (Non-Goals)

- 타임라인 기반 영상 편집, 자막, BGM, 전환 효과
- AI 이미지 생성, 배경 제거 등 사진 보정 기능
- setLog처럼 친구와 협업하는 소셜 기능 (v3+ 고려)
- 피드/DM/팔로우

### 주요 사용자군

- setLog를 써봤지만 "정해진 시간마다 찍는 부담"이 싫은 20대
- 인스타 스토리에 사진 여러 장을 감각적으로 올리고 싶은 사용자
- 쇼핑몰 상세페이지용 비주얼을 빠르게 만드는 1인 셀러
- Canva/미리캔버스를 써봤으나 "템플릿 뻔한 느낌"에 아쉬움을 느낀 사용자

### 차별화 포인트

| 축            | setLog           | PhotoGrid 등     | 본 앱 (v2)                |
| ------------- | ---------------- | ---------------- | ------------------------- |
| 진입          | 정각 촬영        | 템플릿 고르기    | 갤러리에서 사진 고르기    |
| 레이아웃 결정 | 자동 (고정)      | 수동 (프리셋 중) | **자동 제안 + 자유 조정** |
| 영상 편집     | 자동 (제약 있음) | 수동             | 자동 싱크·자동 크롭       |
| 사용자 제어   | 낮음             | 중간             | **높음 (선 드래그)**      |

### 성공 지표 (MVP 기준 KPI)

| 지표                | 목표치 (출시 후 3개월)  | 측정 방법                                         |
| ------------------- | ----------------------- | ------------------------------------------------- |
| 자동 제안 수락률    | 70% 이상                | 제안된 레이아웃을 그대로 저장한 비율              |
| 첫 저장 완료율      | 신규 사용자 중 65% 이상 | 온보딩 이탈 대비 저장 이벤트                      |
| 선 드래그 사용률    | 30% 이상                | 자동 제안을 수정한 사용자 비율 (자유도 니즈 검증) |
| 재방문율 (D7)       | 25% 이상                | 로컬 이벤트 기반 추정                             |
| 영상 포함 저장 비율 | 20% 이상                | 저장 포맷 구분                                    |

## 2. 경쟁 분석

### 직접 경쟁

| 앱              | 강점                | 약점                                         | 우리의 대응                                   |
| --------------- | ------------------- | -------------------------------------------- | --------------------------------------------- |
| setLog          | 편집 0, 바이럴 성공 | 정각 촬영 강제, 협업 필수, 커스터마이징 불가 | 기존 갤러리 사진으로, 혼자서도, 조정 가능하게 |
| BeReal          | 진정성 마케팅       | 2분할 고정                                   | N분할 자유                                    |
| Instagram Edits | Meta 공식 유통      | 본격 영상 편집 방향                          | "한 장 콜라주"에 집중                         |

### 간접 경쟁 (콜라주 앱)

| 앱         | 강점                            | 약점                        |
| ---------- | ------------------------------- | --------------------------- |
| PhotoGrid  | 템플릿 양, 사진+영상 혼합       | 자동 제안 없음, 프리셋 의존 |
| QuickGrid  | 사진+영상 자유 조합, Gen Z 타겟 | 템플릿 중심                 |
| Pic Stitch | 400+ 그리드 프리셋              | 드래그로 셀 크기 조정 불가  |
| Fotor      | 픽셀 단위 간격·마진 조정        | 웹 중심, 모바일 UX 약함     |
| GridArt    | 테두리·배경 커스텀              | 여전히 템플릿 기반          |

### 포지셔닝 맵

```
                    자동 / 편집 0
                         │
              setLog  ●  │
                         │        ● [본 앱]
                         │
  프리셋 ─────────────────┼──────────────── 자유 조정
                         │
             PhotoGrid ● │ ● Pic Stitch
                         │
                    수동 / 편집 多
```

비어있는 우상단("자동 + 자유") 이 본 앱의 포지션.

## 3. 사용자 인터뷰 질문

### 3-1. 워밍업 (5분)

1. 평소에 인스타 스토리나 피드를 얼마나 자주 올리세요?
2. setLog를 써보셨나요? 써봤다면 뭐가 좋았고 뭐가 아쉬웠어요?

### 3-2. 문제 발견 (15분)

1. 여러 장의 사진을 한 화면에 담아 올린 경험이 있으세요? 어떤 도구를 쓰셨어요?
2. 그 작업에서 가장 번거로웠던 단계는 뭐였어요? (사진 고르기 / 레이아웃 / 크롭 / 저장 중)
3. "이 레이아웃이면 됐는데 살짝만 더 조정하고 싶다" 고 느꼈던 적이 있나요?
4. 사진과 영상을 한 화면에 섞어 넣고 싶었던 적 있어요? 그때 어떻게 하셨어요?
5. 콜라주 앱을 쓰다 포기한 적이 있다면, 어떤 순간이었나요?

### 3-3. 행동 관찰 (15분)

1. 지금 핸드폰에서 콜라주를 한번 만들어 보여주세요. (Think-aloud)
2. 만약 앱이 사진 몇 장을 고르자마자 "이런 레이아웃 어때?" 라고 제안하면, 받아들일 것 같으세요 아니면 직접 만들고 싶으세요?
3. 제안된 레이아웃에서 셀 하나만 조금 더 키우고 싶을 때, 어떻게 하면 직관적일까요?

### 3-4. 솔루션 반응 (15분) — 프로토타입 시연

1. 이 앱의 첫인상은요? 어떤 앱 같으세요?
2. 자동 제안 vs 직접 선 긋기, 둘 중 어느 쪽을 더 자주 쓸 것 같아요?
3. 이 앱이 유료라면 얼마까지 지불할 의향이 있으세요?
4. 가장 먼저 있었으면 하는 기능 3가지는?
5. setLog 대신 이 앱을 쓸 것 같은 상황이 있다면?

### 3-5. 마무리 (5분)

1. 이 앱을 어떤 친구에게 가장 먼저 추천하실 것 같아요?
2. 걱정되는 부분이나 아이디어가 있으세요?

## 5. 마이크로카피 초안

**톤 앤 매너:** 친근하지만 군더더기 없음. 자동화 가치를 강조 — "알아서 해줄게" 느낌.

### 5-1. 온보딩 (3 화면, 첫 진입 1회만)

| 화면 | 헤드라인          | 서브 카피                                    | CTA      |
| ---- | ----------------- | -------------------------------------------- | -------- |
| 1/3  | 고르면 알아서     | 사진 몇 장만 골라. 레이아웃은 내가 제안할게. | 다음     |
| 2/3  | 마음에 안 들면 쓱 | 선을 끌면 원하는 크기로 바뀌어.              | 다음     |
| 3/3  | 영상도 자동으로   | 길이 맞추는 것도 다 해줄게.                  | 시작하기 |

### 5-2. 홈 (메인 진입)

- 헤드라인: "오늘은 뭐 모아볼까?"
- Primary CTA: **"사진·영상 고르기"** (크고 명확)
- Secondary: "비율 먼저 정하기" (숨겨진 옵션)
- 최근 작업: "이어서 만들기" (저장 안 한 드래프트 있을 때만)

### 5-3. 자동 제안 화면 (사진 선택 후 등장 — 핵심 순간)

- 헤드라인: "이 중에 마음에 드는 거 있어?"
- 카드 3장 스와이프: "레이아웃 A / B / C"
- 하단 카피: "마음에 안 들면 뒤에서 바꿀 수 있어"
- Primary CTA: **"이걸로"** (선택 시)
- Secondary: "다른 제안 보기" / "빈 캔버스에서 시작"

### 5-4. 에디터 (조정 화면)

| 위치                | 카피                                           |
| ------------------- | ---------------------------------------------- |
| 최초 진입 툴팁      | 선을 끌면 크기가 바뀌어. 눌러서 지울 수도 있어 |
| + 버튼              | 선 추가                                        |
| 셀 탭 (미디어 있음) | 교체 · 삭제 · 꽉 채우기 · 맞추기               |
| 셀 탭 (비어있음)    | 탭해서 사진/영상 넣기                          |
| 선 드래그 중        | 원하는 위치에서 놓으세요                       |
| 선 삭제 확인        | 양옆 칸이 합쳐져. 지울까?                      |
| 되돌리기            | 되돌리기                                       |
| 자동 제안 다시      | 처음으로                                       |

| ------ | --------------------------------- |

### 5-6. 엣지 케이스

- **권한 거부:** "갤러리 접근이 필요해. 설정에서 켜줘."
- **빈 셀로 저장 시도:** "아직 비어있는 칸이 있어. 그대로 저장할까?" [취소 / 저장]
- **사진 1장 선택:** "한 장이면 자르지 않고 그냥 써도 돼. 계속할까?"
- **영상 10개 이상:** "영상이 많으면 저장이 오래 걸려. 괜찮아?"

### 7-1. 메인 플로우 (자동 제안 경로 — 기본)

```

   ↓
[온보딩] (최초 1회)
   ↓
[홈]
   ↓

   ↓ (N장 선택)
[자동 레이아웃 제안]
   ├─ 카드 A · 카드 B · 카드 C (스와이프)
   ├─ [다른 제안 보기] → 추가 생성
   └─ [빈 캔버스] → 수동 경로로 분기
   ↓
[이걸로] 선택
   ↓
[에디터 — 조정 가능]
   ├─ 셀 탭 → 미디어 교체/삭제

   ├─ 선 롱프레스 → 삭제
   └─ + 버튼 → 선 추가
   ↓
[저장]
   ↓
[포맷·비율 선택] → [렌더링] → [갤러리 저장]
```

### 7-2. 수동 플로우 (고급 사용자 경로)

```
[홈]
   ↓
[비율 먼저 정하기] (Secondary CTA)
   ↓
[에디터 — 빈 캔버스]
   ↓
[+ 버튼] → [가로선 / 세로선 추가]
   ↓
[선 이동 + 셀 탭하여 미디어 채우기]
```

**선 삭제:** 선 롱프레스 → "양옆 칸이 합쳐져" 확인 → 병합 (미디어는 더 큰 쪽 유지)

### 9-2. 핵심 로직 설명

### 9-2-1. 자동 레이아웃 제안 알고리즘 (★ 본 앱의 핵심)

**입력:** 사용자가 선택한 미디어 리스트 `[{type: photo|video, aspectRatio: W/H, duration?}]` + 캔버스 비율

**출력:** 3개의 서로 다른 그리드 제안 (V 배열, H 배열, 셀 ↔ 미디어 매핑)

**로직 단계:**

1. **미디어 분류 & 가중치 부여**
   - 영상 = 가중치 1.5 (더 큰 셀에 우선 배치, 주목도 ↑)
   - 가로 사진(aspect > 1.3) = 가로로 긴 셀에 매핑
   - 세로 사진(aspect < 0.8) = 세로로 긴 셀에 매핑
   - 정사각(0.8~1.3) = 어디든 OK
2. **후보 그리드 템플릿 선정**
   - 미디어 개수 N에 따라 규칙 기반으로 후보 3~5개 생성
   - 예: N=4 → "2×2 균등", "좌1 큰 + 우3 작은", "상1 큰 + 하3 작은"
   - 각 템플릿은 V·H 배열의 "소프트 초깃값"을 가짐 (예: V=[0.6], H=[0.5])
3. **최적 매핑 탐색 (미디어 ↔ 셀)**
   - 각 후보 템플릿마다, 셀 종횡비와 미디어 종횡비의 차이 합(loss)을 최소화하는 매핑을 헝가리안 알고리즘(또는 단순 브루트포스 N≤9)으로 계산
   - 영상 가중치를 반영해 영상이 큰 셀로 가도록 편향
4. **상위 3개 제안 선택**
   - loss가 가장 낮은 템플릿 3개 반환
   - 단, 레이아웃 다양성을 위해 "비슷한 구조"는 dedup (분할 방향이 동일하고 선 위치가 ±10% 이내면 중복으로 판단)
5. **"다른 제안 보기"**
   - 이미 보여준 후보를 제외하고 다음 3개 반환 (최대 3회까지)

### 9-2-2. 그리드 데이터 구조

캔버스를 **"수직 분할선 집합 V"**와 **"수평 분할선 집합 H"**로 표현. 각 값은 0.0~1.0의 상대 좌표.

- 예: `V=[0.5]`, `H=[0.3]` → 4개의 셀로 분할
- 셀은 V·H의 교차 영역에서 파생(derived). 직접 저장 ❌
- 셀 ID는 "열 인덱스 + 행 인덱스" 기반 안정적 식별자 → 선 이동 시 미디어가 엉뚱한 셀로 이동 방지

### 9-2-3. 선 이동 제약

- 인접 선(또는 캔버스 경계)과 최소 10% 간격 유지 (너무 좁으면 저장 시 품질 손실)
- 드래그 중 스냅 가이드: 1/2, 1/3, 2/3 지점에서 자석처럼 흡착 (5% 오차 범위)

### 9-2-4. 영상 자동 싱크 (★ 편집 0 철학)

- 셀 중 가장 짧은 영상 길이를 기준 `T_min`으로 설정
- 나머지 영상은 `T_min`까지만 재생 (자동 트림)
- 단, 사용자가 "루프"를 선택한 셀은 `T_min` 동안 반복 재생
- 사진 셀은 `T_min` 동안 고정 프레임
- MVP에서는 `T_min` 최대 15초로 제한 (렌더링 부담)

### 9-2-5. 미디어 렌더링 파이프라인

- **에디터 프리뷰:** Flutter 위젯 트리 (`Stack` + `Positioned` + `ClipRect`)
- **이미지 저장:** `RepaintBoundary` → `ui.Image` → PNG 인코딩
- **영상 저장:** 각 셀의 프레임을 타임라인으로 합성 후 MP4 인코딩 (ffmpeg_kit 또는 네이티브 브릿지)
- 사용자가 비율을 다르게 지정한 경우 letterbox 처리 (MVP) → 후속 버전에서 스마트 크롭
